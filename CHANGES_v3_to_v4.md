# ONA Synthetic Benchmark — v4

An interactive, browser-native tool for generating and visualising synthetic Organisational Network Analysis (ONA) benchmark datasets. Open `ona\_benchmark\_v4.html` in any modern browser. No install, no server, no dependencies.

## A Note on Realism and Applicability

The networks generated by this tool are synthetic by design. They are constructed from statistical models and calibrated parameters, not from observed relational data.

**Topological plausibility, not empirical accuracy.** The topology families (Erdős–Rényi, Watts–Strogatz, Barabási–Albert, SBM, Hierarchical Corporate) produce networks with properties that resemble real organisational networks in specific respects — clustering, hierarchy, hub formation — but any given generated network may not reflect how a real organisation of that size and industry actually operates.

**Synthetic outcomes are benchmarks, not predictions.** The `engagement\_score`, `innovation\_score`, `burnout\_risk`, and `adoption\_velocity` variables are derived from structural position using plausible but simplified assumptions. They are suitable for testing dashboard rendering, method validation, and algorithm benchmarking — not for drawing conclusions about real people or real organisations.

**Industry templates are defaults, not ground truth.** The department mixes, seniority distributions, and location weights in the four industry templates are reasonable approximations, not validated empirical distributions.

**Scenario realism depends on parameter calibration.** The scenarios are tuned to produce structurally interesting networks for specific business questions. Whether any given scenario accurately represents the dynamics of a real reorganisation, merger, or AI adoption programme is a matter of expert judgement, not a guarantee of the tool.

## What Is ONA?

Organisational Network Analysis maps how work actually happens inside a company — who collaborates, who seeks advice, who trusts whom, where silos form, where brokers carry too much load, and how change moves through the organisation. Unlike org-chart analysis, ONA works on the informal relational structure that drives productivity, innovation, and resilience.

Synthetic benchmark datasets allow researchers, practitioners, and product teams to:

* test and compare ONA algorithms on controlled, reproducible ground truth
* demonstrate ONA tooling without using real employee data (privacy-safe)
* model specific business questions before collecting real survey data
* validate platform outputs against known structural properties

## Quick Start

1. Download `ona\_benchmark\_v4.html`
2. Open it in Chrome, Firefox, Safari, or Edge
3. Choose a scenario from the left panel
4. Adjust node count, seeds, and active layers as needed
5. Click **Generate**

No internet connection is required after the file is saved locally.

## Changes from v3

v4 is a rewrite of the core generation algorithms. The user interface, export contracts, scenario IDs, and layer model are unchanged. The changes below are all internal to the generation pipeline.

### BA topology — power-law degree distribution

**v3 problem:** The preferential attachment inner loop recomputed the full weight sum `Σ degree^1.2` on every candidate trial — O(n²) per arriving node. With `m=3` edges per node and a mild power exponent (1.2), the resulting degree distribution was nearly Poisson: no visible fat tail, no hubs.

**v4 fix:** Replaced with a stochastic acceptance sampler (O(1) per trial): sample a random candidate `j`, accept with probability `w\[j] / w\_max`. Increased to `m=5`, power `1.3`, zero appeal `1.0`. Seniority fitness multipliers (executive = 3.5×, individual = 1.0×) are applied directly to the weight array during construction, so executives become hubs organically rather than via a post-hoc rewire. Post-construction rewiring increased from 10% to 15%, targeting low-fitness peripheral nodes.

### HierCorp topology — star hierarchy

**v3 problem:** A separate tree was built per department with no cross-department root. The global CEO node did not exist. `Math.min/max` on node indices was used as the edge key, which lost directionality. Within-department informal ties used `0.18 × choose2(n)` — for a 100-person department this produces \~900 edges, making every department a near-clique. Span of control was uniform random (4–8), with no differentiation between executive and IC layers.

**v4 fix:** Single global CEO node identified by highest level + tenure. Dept VPs report to CEO. Span-of-control is level-dependent (exec: 3–4 spans, IC managers: 10–12). Informal within-dept ties capped at `min(√dept\_size × 2, 40)` edges, drawn with seniority-biased acceptance probability. Cross-dept leadership bridges added between level ≥ 4 nodes (\~8% of senior count).

### SBM topology — degree heterogeneity

**v3 problem:** Flat block probability `p\_ij = block\[bi]\[bj]` for all node pairs produced a Poisson degree distribution within each block. Every node in a block had roughly the same expected degree.

**v4 fix:** Degree-corrected SBM. Each node receives a sociability score `θ\_i` drawn from a Pareto distribution (α=1.5, x\_min=0.5), clipped and multiplied by a seniority factor (executive = 2.6×). Connection probability becomes `p\_ij = block\[bi]\[bj] × θ\_i × θ\_j`, normalised per block. This preserves block structure while producing a fat-tailed degree distribution within each block.

### Layer derivation — hub structure preservation

**v3 problem:** Uniform keep-probability per edge (e.g. `trust: 0.54`) applied identically to all edges. A hub with 40 edges and an isolated node with 6 edges both lost the same proportion of ties in each layer. The heavy tail of the base graph was not preserved in any derived layer.

**v4 fix:** Keep-probability scales with `max(degree\_i, degree\_j) / max\_degree` via a layer-specific hub-boost multiplier. Communication layer: boost 0.60. Mentorship: boost 0.20. High-degree nodes retain their centrality position across all derived layers independently.

### DEMO\_M silo calibration

**v3 problem:** Between-block probability set to 0.005 produced two near-disconnected cliques — visually two separate networks with virtually no cross-department edges. This is not a realistic silo finding; it is a broken graph.

**v4 fix:** Between-block raised to 0.02. Produces a realistic E-I index of approximately −0.25 to −0.35 — a visible but tractable silo, suitable for a first-meeting demonstration.

### Validation

Added a **Degree Variance-to-Mean Ratio (VMR)** check to the validation overlay. VMR > 1.5 confirms non-Poisson (power-law-like) degree distribution. For Erdős–Rényi this is \~1.0 by design; for BA and HierCorp it should be 3–20+. This check exposes flat distributions that previously passed all other validation criteria.
